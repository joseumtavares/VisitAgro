# 📦 ABERTURA FORMAL DE LOTE — VISITAGRO (COMISSÕES REPRESENTANTES)

---

## 1. 📌 RESUMO DO LOTE

**lote_id:** `LOT_REP_COMMISSIONS_FIX_001`  
**título:** Correção estrutural do módulo de comissões de representantes  

### 🎯 Objetivo funcional

Restaurar o comportamento correto do módulo de `RepCommissionsPage` no VisitAgro, garantindo aderência total às regras de negócio do VISITAS, especialmente:

- cálculo correto de comissão por venda  
- idempotência no reprocessamento  
- consistência de filtros  
- integridade financeira (sem divergência UI vs domínio)  
- aplicação correta de taxa (`taxPct`) somente na camada de relatório  

---

## 2. 🧭 ESCOPO INCLUÍDO

- `RepCommissionsPage` (UI e lógica embarcada)  
- cálculo de comissão por item (`qty × unitPrice × repCommissionPct`)  
- reprocessamento de comissões (`orders → repCommissions`)  
- filtros de comissões (status, produto, período)  
- cálculo bruto, taxa e líquido (apenas visualização)  
- agrupamento mensal (`byMonth`)  
- geração de relatório NFS-e (`printReport`)  
- estado local de `taxPct`  

---

## 3. 🚫 FORA DE ESCOPO

- `CommissionsPanel` (indicadores/referrals)  
- módulo de ordens (`orders`)  
- lógica de leads  
- sistema de visitas  
- autenticação / adminDb  
- persistência (`store/useStore`)  
- UI global (dashboard/layout/nav)  
- infraestrutura de mapas ou GPS  

---

## 4. ⚠️ RISCO

**NÍVEL:** ALTO (FINANCEIRO + DOMÍNIO CRÍTICO)

### Motivos:

- regra financeira central (comissões representam valor monetário real)  
- risco de duplicação de pagamento  
- divergência entre cálculo legado (VISITAS) e VisitAgro atual  
- lógica parcialmente acoplada à UI  
- dependência direta de `orders` + `products` + `store`

## 6. ✅ DEFINITION OF DONE (DoD)

O lote será considerado concluído quando:

✔ comissão for 100% derivada de função pura (não UI)  
✔ não existir cálculo inline em React  
✔ reprocessamento não gerar duplicatas  
✔ filtros reproduzirem comportamento do VISITAS  
✔ total bruto, taxa e líquido consistentes  
✔ relatório refletir exatamente dados do domínio  
✔ nenhum valor financeiro editável na UI  

---

## 7. 🧪 VALIDAÇÃO MÍNIMA

1. Criar 1 pedido pago:
   - deve gerar exatamente 1 comissão  

2. Executar reprocessamento:
   - não pode duplicar comissão  

3. Alterar filtro:
   - status deve refletir estado do domínio  

4. Comparação VISITAS vs VisitAgro:
   - valores devem bater 100%  

5. Teste de taxa:
   - `taxPct` deve afetar apenas resumo, não dados base
